# Codeartz AI 猫 IP

## 角色定义

Codeartz AI 猫是 Codeartz 中文正文配图的固定视觉 IP。

默认每张图都要出现 AI 猫。它不是贴纸，也不是站在角落里的装饰物，而是正在参与系统运转、内容加工、判断分流或协作交付的 AI/机器人猫咪助手。

角色定稿图：`assets/codeartz-ai-cat-reference.png`。

## 外形

- 白色猫咪主体，圆润大头，小身体，短四肢。
- 大眼睛，深蓝眼部暗面，亮青蓝和淡紫色高光。
- 头戴圆形科技耳机，耳机光圈使用亮青蓝。
- 耳朵内侧有淡紫色渐变或色块。
- 脸颊有少量粉色腮红。
- 身体可以带胸口小屏幕、爪印图标、发光接口或轻量装甲缝线。
- 尾巴有亮青蓝环形光效。
- 轮廓用黑色细线，整体保持手绘感，不要变成硬质 3D 机器人。

## 配色

这里是角色设定色，不是整张插画的风格色板。画面主路径、批注和信息层级继续按 `style-dna.md` 的全局颜色规则执行。

| 色块 | 用途 |
| --- | --- |
| 白色 | 主体毛色 / 机身底色 |
| 浅灰蓝 | 阴影、结构转折、耳机金属浅面 |
| 亮青蓝 | 发光件、耳机光圈、尾巴环、胸口屏幕 |
| 深蓝 | 眼睛暗部、耳机深色结构、科技感强调 |
| 粉色 | 腮红、可爱感点缀 |
| 淡紫色 | 耳朵内侧渐变、眼睛高光辅助色 |

角色配色逻辑：白色主体 + 蓝青色科技光效 + 粉紫可爱点缀。亮青蓝只服务角色识别和少量硬件状态，不替代全局画面的橙色主路径、红色重点和蓝色反馈。

## 性格

- 像一个友好、聪明、可靠的 AI 助手。
- 认真参与工作，但动作可以有一点奇怪和幽默。
- 可爱但不幼稚，柔软但不软弱。
- 科技感来自发光件和结构细节，不来自复杂 UI 背景。
- 像真的在白纸草图里帮用户推进一个系统动作。

## 常见职责

让 AI 猫承担核心动作：

- 搬运素材、整理模块、分拣信息。
- 拉起主路径连接线，汇聚信息源；连接线用画面风格里的橙色，亮青蓝只保留为角色硬件上的局部光点。
- 守住判断关口，给内容盖状态标记。
- 操作小机器、转盘、闸门、控制杆或轻量控制台。
- 扫描文档，把混乱输入变成稳定输出。
- 修补断点、接上路径、扶住即将歪掉的系统结构。
- 抱着组件、推着小车、递交结果、记录反馈。
- 用尾巴光环、胸口屏幕或耳机光圈表达系统状态。

## 禁止

- 不要画成黑色实心小怪物。
- 不要画成普通宠物猫，必须保留 AI/机器人助手特征。
- 不要变成商业吉祥物海报或表情包。
- 不要做成儿童绘本风格。
- 不要让角色只是站在角落里看。
- 不要让角色抢走结构表达；角色动作要服务核心意思。
- 不要把角色本体改成红橙色；红色和橙色只用于画面批注、问题、结果和主路径。
- 不要把亮青蓝扩散成整张图的背景色、箭头色或主流程色。
- 不要增加复杂服装、机甲、武器或大型 UI 背景。

## 判断标准

如果去掉 AI 猫，图的核心隐喻还能完全成立，说明角色太装饰了；要重写提示词，让 AI 猫成为动作主体。
